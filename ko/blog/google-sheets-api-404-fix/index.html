구글 Sheets API 404 (Not Found) 에러와 진짜 원본 스프레드시트 ID 발굴

가짜 해시 ID의 트랩

스프레드시트 API를 연동할 때, 구글 드라이브 내의 '공유' 혹은 '웹에 게시' 주소 끝에 부착된 문자열을 고유 ID 키로 사용하면 404 Not Found API 호출 오류가 뜹니다.

진짜 스프레드시트 ID 식별

브라우저 주소창에 나타나는 /d/와 /edit 사이의 고유 원본 문자열(ID)을 추출하여 API 키로 이식해야 합니다. 추가적으로 구글 클라우드 플랫폼에서 생성한 서비스 어카운트(Service Account) 이메일 주소를 스프레드시트의 '편집자 공유 대상'으로 추가하여 접근 권한 문제를 완전히 해결했습니다.

연관도 계산 방식 알아보기

본 포트폴리오의 추천 콘텐츠는 Google Gemini API(gemini-embedding-2 모델)를 활용해 추출한 다차원 텍스트 임베딩 벡터 간의 코사인 유사도(Cosine Similarity)를 기준으로 정렬됩니다.

글의 제목, 카테고리, 요약 설명 및 전체 본문을 종합 분석하여 단순한 키워드 일치를 넘어선 의미론적 연관성(Semantic Similarity)을 판별합니다. API 호출이 불가한 환경에서는 로컬 TF-IDF 기반의 단어 빈도 분석 알고리즘으로 자동 전환(Fallback)되어 연관도를 계산합니다.
