미디어 쿼리의 한계점

미디어 쿼리는 화면 전체 크기(Viewport Width)만 알 수 있습니다. 이 때문에 사이드바가 열려 본문 영역이 좁아지는 등 컴포넌트가 놓이는 실제 맥락의 크기 변화에는 반응하지 못해 디자인이 꼬이게 됩니다.

Container Queries (Baseline 2023)

부모 요소에 container-type: inline-size를 선언하면, 자식 요소들은 @container (min-width: 400px)와 같이 부모의 크기를 직접 쿼리하여 스스로 외형을 결정합니다. 이를 통해 어떤 화면 위치에 이식되더라도 완벽하게 적응하는 모듈형 컴포넌트 구현이 가능해집니다.

연관도 계산 방식 알아보기

본 포트폴리오의 추천 콘텐츠는 Google Gemini API(gemini-embedding-2 모델)를 활용해 추출한 다차원 텍스트 임베딩 벡터 간의 코사인 유사도(Cosine Similarity)를 기준으로 정렬됩니다.

글의 제목, 카테고리, 요약 설명 및 전체 본문을 종합 분석하여 단순한 키워드 일치를 넘어선 의미론적 연관성(Semantic Similarity)을 판별합니다. API 호출이 불가한 환경에서는 로컬 TF-IDF 기반의 단어 빈도 분석 알고리즘으로 자동 전환(Fallback)되어 연관도를 계산합니다.
