맥북 수명 단축 없는 고속 AI 추론: 로컬 RAM과 SSD 스왑 최적화

SSD 스왑 마모의 공포

가벼운 노트북에서 로컬 AI를 무리하게 가동하면, OS는 부족한 RAM을 보완하기 위해 SSD의 일부를 가상 메모리로 강제 할당하는 '스왑(Swap) 연산'을 끝없이 수행합니다. 이는 SSD의 읽기/쓰기 수명(TBW)을 파괴하는 주원인입니다.

하드웨어 친화적 아키텍처

로컬 기기에서는 텍스트 편집기(Obsidian)와 가벼운 인터페이스만 가동하고, 코어 추론은 구글 Gemini API 서버로 이원화합니다. 이렇게 하면 로컬 CPU/GPU 연산 점유율이 0%에 수렴하므로 맥북의 쿨링 팬이 돌 일이 없으며, 물리적인 SSD 스왑 유발을 완벽히 방지하여 맥북의 하드웨어 수명을 극대화할 수 있습니다.

연관도 계산 방식 알아보기

본 포트폴리오의 추천 콘텐츠는 Google Gemini API(gemini-embedding-2 모델)를 활용해 추출한 다차원 텍스트 임베딩 벡터 간의 코사인 유사도(Cosine Similarity)를 기준으로 정렬됩니다.

글의 제목, 카테고리, 요약 설명 및 전체 본문을 종합 분석하여 단순한 키워드 일치를 넘어선 의미론적 연관성(Semantic Similarity)을 판별합니다. API 호출이 불가한 환경에서는 로컬 TF-IDF 기반의 단어 빈도 분석 알고리즘으로 자동 전환(Fallback)되어 연관도를 계산합니다.
