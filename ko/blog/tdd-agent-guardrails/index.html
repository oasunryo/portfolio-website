인간의 개입을 줄이는 Guardrail 설계

코드를 받고 나서 사람이 직접 실행해보고 에러를 발견한 뒤 다시 프롬프트를 작성하는 것은 많은 토큰과 시간의 낭비를 부릅니다. AI 스스로 검증할 수 있는 피드백 루프가 핵심입니다.

TDD 기반의 에이전트 검증 루프

에이전트에게 "기능 구현 전 단위 테스트 코드를 먼저 작성하고, 터미널 명령어를 통해 그 테스트가 완전하게 통과하는지 확인한 로그를 포함해 결과를 출력하라"는 하네스 제약을 줍니다. AI는 스스로 테스트 오류를 보고 스스로 리팩토링하는 과정을 반복해 무결점 코드를 제출하게 됩니다.

연관도 계산 방식 알아보기

본 포트폴리오의 추천 콘텐츠는 Google Gemini API(gemini-embedding-2 모델)를 활용해 추출한 다차원 텍스트 임베딩 벡터 간의 코사인 유사도(Cosine Similarity)를 기준으로 정렬됩니다.

글의 제목, 카테고리, 요약 설명 및 전체 본문을 종합 분석하여 단순한 키워드 일치를 넘어선 의미론적 연관성(Semantic Similarity)을 판별합니다. API 호출이 불가한 환경에서는 로컬 TF-IDF 기반의 단어 빈도 분석 알고리즘으로 자동 전환(Fallback)되어 연관도를 계산합니다.
